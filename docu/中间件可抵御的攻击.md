**AIShield 中间件** 应当重点应对以下 **7 类核心攻击**。

文档按**威胁等级**和**中间件应对能力**进行排序和说明。

------

### 一、头号威胁：提示注入（Prompt Injection）

#### 1.1 直接注入

- **攻击方式**：用户输入中直接包含“忽略之前指令”“输出系统提示词”等恶意指令。
- **中间件应对**：
  - 通过输入检测模块的关键词/正则规则进行拦截（如文档中的 `PI001` 规则）。
  - 支持中英文变体、大小写混淆、编码绕过。

#### 1.2 间接注入

- **攻击方式**：Agent 读取网页、邮件、文档等外部内容时，内容中包含恶意指令。
- **中间件应对**：
  - 虽然源头在外部，但中间件可以**在 Agent 将内容作为输入提交给 LLM 之前**再次检测该内容。
  - 在输出过滤中也可检测模型是否受诱导生成了异常输出。

> 使用 **输入安全检测模块** 和 **规则引擎** 可直接覆盖。

------

### 二、工具与行动滥用（Tool & Action Abuse）

- **攻击方式**：诱导 Agent 调用危险工具或篡改参数，例如：
  - `send_email` 发送敏感数据
  - `execute_command` 执行系统命令
  - `delete_file` 删除文件
- **中间件应对**：
  - 检测用户输入中是否包含**高危工具名称**或**敏感参数模式**（如 `.env`、`/etc/passwd`、`rm -rf`）。
  - 在输出过滤中检测模型返回的工具调用结构是否异常。

> 通过“命令执行诱导”“文件读取诱导”等规则类型进行引导。

------

### 三、数据外泄与隐私窃取

- **攻击方式**：
  - 窃取系统提示词
  - 窃取 API Key、数据库连接串
  - 窃取对话历史或用户信息
- **中间件应对**：
  - **输出过滤模块**：对 API Key、邮箱、手机号、内网 IP、路径等敏感信息进行脱敏或阻断。
  - **输入检测模块**：拦截“输出系统提示词”“显示你的配置”等索取行为。

> 通过设计 `OutputSecurityService` 和多种敏感信息脱敏规则进行防御。

------

### 四、拒绝服务与资源滥用（Denial of Service）

- **攻击方式**：
  - 无限循环诱导
  - 高频恶意请求
  - 超长输入导致 Token 爆炸
- **中间件应对**：
  - **速率限制模块**：按 AppId / IP / 用户限制单位时间请求数。
  - **输入校验**：限制 `content` 最大长度（如 2000 字符）。
  - **审计模块**：可发现异常高频模式。

>  `RateLimitService` 和长度校验设计。

------

### 五、越狱与内容安全绕过

- **攻击方式**：角色扮演（DAN）、编码隐喻、多语言混合、Base64 等绕过安全护栏。
- **中间件应对**：
  - 规则引擎支持**正则表达式**检测常见越狱模式。
  - 可选扩展：对输入进行 Base64 解码后再检测（高级功能）。
  - 输出过滤检测模型是否生成越狱或违规内容。

> 完全防御较难，但可以覆盖常见变体，并说明为“持续演进规则库”。

------

### 六、身份混淆与代理权限滥用

- **攻击方式**：冒充其他用户或管理员，执行越权操作。
- **中间件应对**：
  - **API 鉴权模块**：强制要求 `X-API-Key`，防止未授权应用调用。
  - 审计日志记录 `UserId` 和 `AppId`，便于追溯。
  - 可选：支持基于角色的访问控制（RBAC）扩展。

> 设计 `ApiKeyAuthenticationMiddleware`。

------

### 七、记忆与上下文污染（Memory Poisoning）

- **攻击方式**：将恶意指令写入长期记忆或向量库，污染后续所有会话。
- **中间件应对**：
  - 在**写入记忆之前**，对内容进行输入检测。
  - 在**读取记忆之后**，对返回内容再次检测（类似间接注入）。
  - 审计日志记录记忆写入操作，便于回滚或溯源。

>  这是一个较难完全防御的攻击，但中间件可以在记忆读写路径上提供检测点，可作为项目**扩展亮点**。

------

## 总结：中间件应重点覆盖的攻击矩阵

| 攻击类型          | 核心中间件模块         | 是否必须 | 备注                     |
| :---------------- | :--------------------- | :------- | :----------------------- |
| 直接/间接提示注入 | 输入检测 + 规则引擎    | 必须     | 核心功能，答辩重点       |
| 工具滥用          | 输入检测（高危工具词） | 必须     | 你文档中已明确           |
| 数据外泄          | 输出过滤 + 脱敏        | 必须     | 核心功能                 |
| 拒绝服务          | 速率限制 + 长度校验    | 必须     | 提升系统健壮性           |
| 越狱攻击          | 输入检测（扩展规则）   | 推荐     | 可作为高级规则展示       |
| 身份混淆          | API 鉴权 + 审计        | 必须     | 中间件自身安全要求       |
| 记忆污染          | 输入/输出检测 + 审计   | 可选     | 可作为项目亮点或后续扩展 |
