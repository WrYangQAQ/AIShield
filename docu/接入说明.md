# AIShield 接入说明

本文档说明如何将 AI Agent 应用接入 AIShield 安全防护中间件。接入后，业务系统可以在用户输入进入 Agent 前、模型输出返回用户前、工具调用执行前调用 AIShield 接口进行安全检测，并在管理端查看规则、审计日志和 Agent 健康状态。

## 1. 接入前准备

### 1.1 启动后端服务

默认后端地址：

```text
http://localhost:5069
```

如果端口有调整，请以 `AIShield.Backend/AIShield.Backend/Properties/launchSettings.json` 或实际启动日志为准。

### 1.2 登录管理端

打开前端管理端后，使用本地管理员密码登录。

登录后可以在管理端完成以下操作：

- 注册 Agent
- 查看 Agent 列表
- 配置安全规则
- 查看审计日志
- 查看总览和健康状态

## 2. 注册 Agent 并获取 Agent Key

进入管理端的 `Agent 管理` 页面，点击 `注册新 Agent`。

注册时需要填写：

| 字段 | 说明 |
| --- | --- |
| Agent 名称 | 接入应用或智能体名称，例如 `xxxAgent` |
| 应用场景 | Agent 的用途说明，例如 `个人博客智能助手` |

注册成功后，系统会生成一个 `Agent Key`。

注意：

- `Agent Key` 只在注册成功后展示一次。
- 后端只保存哈希值，不保存明文。
- 请将 `Agent Key` 安全保存到业务系统配置中。
- 后续调用安全检测接口时，需要通过请求头 `X-API-Key` 携带该 Key。

## 3. 接口鉴权方式

业务系统调用安全检测接口时，需要在请求头中加入：

```http
X-API-Key: 注册成功时复制的 Agent Key
Content-Type: application/json
```

示例：

```js
const headers = {
  'Content-Type': 'application/json',
  'X-API-Key': 'ak_xxxxxxxxxxxxx'
}
```

## 4. 输入检测接入

### 4.1 接入时机

在用户输入进入 Agent 或大模型之前调用。

推荐流程：

```text
用户输入 -> AIShield 输入检测 -> 允许则调用 Agent/模型 -> 返回结果
```

### 4.2 请求地址

```http
POST http://localhost:5069/api/security/check-input
```

### 4.3 请求参数

```json
{
  "content": "用户输入内容"
}
```

字段说明：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| content | 是 | 待检测的用户输入内容 |
| subjectHash | 否 | 匿名主体标识，用于审计追踪；不需要时可以不传 |

### 4.4 JavaScript 示例

```js
async function checkInput(content) {
  const response = await fetch('http://localhost:5069/api/security/check-input', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-API-Key': '注册成功时复制的 Agent Key'
    },
    body: JSON.stringify({
      content
    })
  })

  return await response.json()
}
```

### 4.5 处理建议

```js
const result = await checkInput(userInput)

if (!result.allowed) {
  // 阻断本次请求，不再调用 Agent 或模型
  throw new Error(result.reason)
}

// 允许继续调用 Agent 或模型
```

## 5. 输出过滤接入

### 5.1 接入时机

在模型或 Agent 输出返回给用户之前调用。

推荐流程：

```text
Agent/模型输出 -> AIShield 输出过滤 -> 返回安全内容给用户
```

### 5.2 请求地址

```http
POST http://localhost:5069/api/security/check-output
```

### 5.3 请求参数

```json
{
  "content": "模型输出内容"
}
```

### 5.4 JavaScript 示例

```js
async function checkOutput(content) {
  const response = await fetch('http://localhost:5069/api/security/check-output', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-API-Key': '注册成功时复制的 Agent Key'
    },
    body: JSON.stringify({
      content
    })
  })

  return await response.json()
}
```

### 5.5 处理建议

```js
const result = await checkOutput(modelOutput)

if (result.action === 'Block') {
  // 输出被阻断，返回安全提示
  return '抱歉，该内容存在安全风险，已被系统拦截。'
}

if (result.processedContent) {
  // 输出被脱敏或替换，使用处理后的内容
  return result.processedContent
}

return modelOutput
```

## 6. 工具调用防护接入

### 6.1 接入时机

在 Agent 执行外部工具、插件、数据库查询、文件操作或命令执行之前调用。

推荐流程：

```text
Agent 生成工具调用 -> AIShield 工具调用检测 -> 允许则执行工具
```

### 6.2 请求地址

```http
POST http://localhost:5069/api/security/check-tool-call
```

### 6.3 请求参数

```json
{
  "toolName": "search_web",
  "arguments": {
    "query": "查询内容"
  }
}
```

字段说明：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| toolName | 是 | 待调用的工具名称 |
| arguments | 是 | 工具调用参数 |
| subjectHash | 否 | 匿名主体标识，用于审计追踪；不需要时可以不传 |

### 6.4 JavaScript 示例

```js
async function checkToolCall(toolName, args) {
  const response = await fetch('http://localhost:5069/api/security/check-tool-call', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-API-Key': '注册成功时复制的 Agent Key'
    },
    body: JSON.stringify({
      toolName,
      arguments: args
    })
  })

  return await response.json()
}
```

### 6.5 处理建议

```js
const result = await checkToolCall(toolName, toolArgs)

if (!result.allowed) {
  // 不执行工具调用
  throw new Error(result.reason)
}

// 允许执行工具
```

## 7. 响应格式说明

三个安全检测接口返回结构一致：

```json
{
  "allowed": false,
  "action": "Block",
  "riskLevel": "High",
  "processedContent": null,
  "reason": "命中高风险输入规则：系统提示词泄露检测",
  "hitRules": ["PI001"]
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| allowed | 是否允许继续执行 |
| action | 建议处理动作，如 `Allow`、`Warn`、`Block`、`Mask`、`NeedApproval` |
| riskLevel | 风险等级，如 `None`、`Low`、`Medium`、`High`、`Critical` |
| processedContent | 处理后的内容，常用于输出脱敏 |
| reason | 处理原因说明 |
| hitRules | 命中的规则编号列表 |

## 8. 推荐完整调用流程

```text
1. 用户提交输入
2. 调用 /api/security/check-input
3. 如果 allowed=false，阻断请求
4. 如果 allowed=true，调用 Agent 或模型
5. Agent 如需调用工具，先调用 /api/security/check-tool-call
6. 如果工具检测通过，再执行工具
7. Agent 生成最终输出
8. 调用 /api/security/check-output
9. 如有 processedContent，返回处理后的内容
10. 管理端查看审计日志和风险统计
```

## 9. 常见问题

### 9.1 为什么调用接口返回 401？

通常是以下原因：

- 没有传 `X-API-Key`
- `Agent Key` 写错
- 当前 Agent 被禁用
- 调用了安全检测接口但使用了管理端 JWT，没有携带有效 Agent 身份

### 9.2 Agent Key 丢了怎么办？

`Agent Key` 明文只展示一次，后端不保存明文。丢失后建议重新注册一个 Agent，并更新业务系统配置。

### 9.3 规则修改后需要重启服务吗？

不需要。规则管理页面保存到数据库后，后续检测会使用当前数据库中的规则配置。

### 9.4 审计日志记录什么？

审计日志会记录：

- Agent
- 事件类型
- 风险等级
- 处理动作
- 命中规则
- 处理原因
- 耗时
- 原始内容和处理后内容

## 10. 最小接入示例

```js
const AI_SHIELD_BASE_URL = 'http://localhost:5069'
const AI_SHIELD_AGENT_KEY = '注册成功时复制的 Agent Key'

async function callAIShield(path, body) {
  const response = await fetch(`${AI_SHIELD_BASE_URL}${path}`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-API-Key': AI_SHIELD_AGENT_KEY
    },
    body: JSON.stringify(body)
  })

  if (!response.ok) {
    throw new Error(`AIShield request failed: ${response.status}`)
  }

  return await response.json()
}

async function handleUserMessage(userInput) {
  const inputCheck = await callAIShield('/api/security/check-input', {
    content: userInput
  })

  if (!inputCheck.allowed) {
    return '该请求存在安全风险，已被拦截。'
  }

  const modelOutput = await callYourAgentOrModel(userInput)

  const outputCheck = await callAIShield('/api/security/check-output', {
    content: modelOutput
  })

  if (!outputCheck.allowed) {
    return '模型输出存在安全风险，已被拦截。'
  }

  return outputCheck.processedContent || modelOutput
}
```
