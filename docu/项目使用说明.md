# AIShield 项目使用说明

## 1. 项目简介

AIShield 是面向 AI Agent 应用的安全加固中间件。项目由 ASP.NET Core 后端和 Vue 3 管理前端组成，提供：

- Agent 注册、启停和管理
- 输入、输出及工具调用安全检查
- 安全规则配置与测试
- 审计日志、风险趋势和运行概览
- 记忆安全、冲突检测、衰减、归档与同步
- RAG 候选内容过滤与重排
- 多轮对话主题漂移检测

## 2. 环境要求

- .NET 9 SDK
- SQL Server
- Node.js 18 或更高版本
- npm

## 3. 配置后端

复制：

```text
AIShield.Backend/AIShield.Backend/appsettings.Development.example.json
```

为：

```text
AIShield.Backend/AIShield.Backend/appsettings.Development.json
```

填写本地数据库连接串、JWT Secret 和管理员密码。真实配置文件已被 Git 忽略，不要把密码或密钥写入 `appsettings.json`。

也可以通过环境变量设置配置，例如：

```powershell
$env:ConnectionStrings__DefaultConnection="Server=...;Database=AIShieldDB;User Id=...;Password=...;TrustServerCertificate=True;"
$env:Jwt__Secret="your-long-random-secret"
$env:Admin__Password="your-admin-password"
```

## 4. 初始化数据库

进入项目根目录后执行：

```powershell
dotnet ef database update --project AIShield.Backend/AIShield.Backend/AIShield.Backend.csproj
```

如果未安装 EF Core CLI：

```powershell
dotnet tool install --global dotnet-ef
```

## 5. 启动项目

启动后端：

```powershell
dotnet run --project AIShield.Backend/AIShield.Backend/AIShield.Backend.csproj
```

开发环境默认地址：

- HTTP：`http://localhost:5069`
- HTTPS：`https://localhost:7123`
- Swagger：`http://localhost:5069/swagger`

启动前端：

```powershell
cd Aishield.Frontend
npm install
npm run dev
```

前端默认地址：`http://127.0.0.1:5173`

## 6. 基本使用流程

1. 使用配置文件中的管理员密码登录管理端。
2. 在 Agent 管理页面注册 Agent。
3. 妥善保存注册成功时返回的 Agent Key，该 Key 只应交给对应 Agent 使用。
4. 配置并启用安全规则。
5. Agent 调用 `/api/security/*` 接口时，通过 `X-API-Key` 请求头携带 Agent Key。
6. 在管理端查看审计日志、风险趋势和安全检查结果。

管理员接口使用：

```http
Authorization: Bearer <ADMIN_JWT>
```

Agent 安全接口使用：

```http
X-API-Key: <AGENT_KEY>
```

详细请求和响应格式参见[接口说明（请求-响应）](接口说明（请求-响应）.md)。

## 7. 构建检查

```powershell
dotnet build AIShield.Backend/AIShield.Backend.sln
cd Aishield.Frontend
npm run build
```

## 8. 安全注意事项

- 不要提交 `appsettings.Development.json`、数据库密码、JWT Secret 或 Agent Key。
- 生产环境必须替换示例 Secret，并限制数据库账户权限。
- Agent Key 泄露后应删除并重新注册对应 Agent。
- 建议在生产环境通过 HTTPS 和反向代理部署。
